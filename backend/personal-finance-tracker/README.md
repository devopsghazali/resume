# Personal Finance Tracker API

Simple Node.js backend for income, expense, and category tracking.

## Project

The foundation project focuses on request flow, validation, auth, and clean service separation.

## Stack

- Node.js
- Express
- PostgreSQL
- Zod
- JWT

## Questions

1. How do you validate finance input before saving it?
2. How do you protect user-owned records?
3. How do you model categories and transactions cleanly?
4. How do you make update routes idempotent?
5. How do you handle totals and summaries efficiently?
6. How do you separate controller and service logic?
7. How do you store user context per request?
8. How do you test a CRUD API quickly?
9. How do you keep the API simple but production-minded?
10. How do you explain this project in an interview?
